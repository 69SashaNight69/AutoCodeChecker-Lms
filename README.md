# AutoCodeChecker Pro — LMS-платформа автоматичної перевірки програмних завдань

**AutoCodeChecker Pro** — це хмароорієнтована (Serverless) веб-платформа (LMS) для автоматизованого оцінювання та інтелектуального аналізу програмних завдань на мові C#. Платформа дозволяє студентам писати код у веб-редакторі, отримувати миттєву оцінку правильності (на базі тест-кейсів), метрики швидкодії, а також розгорнутий текстовий аналіз якості коду від штучного інтелекту OpenAI.

## 🛠️ Необхідне супровідне програмне забезпечення (Prerequisites)

### 1. Системні вимоги (ОС та Середовище)
*   **Операційна система:** Windows 10/11 (із увімкненою віртуалізацією в BIOS та компонентом WSL 2) або Linux / macOS.
*   **Docker & Docker Desktop:** Для швидкого контейнеризованого розгортання всієї інфраструктури локально.
*   **Платформа .NET 8.0 SDK:** Необхідна для локального збирання, запуску міграцій та виконання автоматичних тестів.
*   **Node.js (версія LTS):** Необхідна для запуску та збирання клієнтської частини (фронтенду).
*   **pgAdmin 4:** Утиліта для зручного адміністрування та перегляду таблиць бази даних.

### 2. Ключові бібліотеки Бекенду (C# / .NET)
*   `Microsoft.CodeAnalysis.CSharp.Scripting` (Roslyn API) — для динамічної компіляції та виконання коду в пам'яті.
*   `Npgsql.EntityFrameworkCore.PostgreSQL` — для взаємодії з реляційною БД PostgreSQL.
*   `BCrypt.Net-Next` — для надійного криптографічного хешування паролів.
*   `Microsoft.AspNetCore.Authentication.JwtBearer` — для авторизації за допомогою токенів (JWT).
*   `Amazon.Lambda.AspNetCoreServer.Hosting` — міст для роботи Minimal API в безсерверному середовищі AWS Lambda.

### 3. Ключові бібліотеки Фронтенду (React)
*   `@monaco-editor/react` — інтерактивний редактор коду з підсвіткою синтаксису (аналог VS Code).
*   `react-markdown` — для рендерингу умов завдань, що підтримують Markdown-форматування (текст, таблиці, зображення).
*   `html2pdf.js` — для автоматичної генерації та завантаження красивих PDF-звітів успішності українською мовою.

## 📦 Швидкий запуск через Docker Compose (Локальний Development)

Усі компоненти системи локально налаштовані на роботу в єдиній закритій мережі в безпечному діапазоні портів **7000–7003** (7000 — API, 7002 — React UI, 7003 — PostgreSQL).

### Покроковий алгоритм запуску:

1.  **Налаштуйте секрети:** Відкрийте файл `appsettings.json` (в папці `AutoCodeChecker.LocalApi`) та переконайтеся, що там вказано ваш діючий ключ **OpenAIApiKey** (OpenAI API) та рядок підключення до бази.
2.  **Запустіть Docker Desktop** на комп'ютері.
3.  **Відкрийте термінал** у кореневій папці проекту (там, де файл `docker-compose.yml`) та виконайте команду:
    ```bash
    docker-compose up --build -d
    ```
4.  **Результат запуску:**
    *   **Локальний UI:** [http://localhost:7002](http://localhost:7002)
    *   **Локальний API (Swagger):** [http://localhost:7000/swagger](http://localhost:7000/swagger)
    *   **Локальна БД:** доступна на порту **`7003`** (користувач: `postgres`, база: `autocodechecker`).
